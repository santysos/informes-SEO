01Setup de plugins

Limpia primero, instala después. La instalación actual tiene basura inactiva que ralentiza todo.

Desactivar y eliminar

Desde WP Admin → Plugins elimina (no solo desactivar): AirLift, Akismet, Ally, Anti-Malware, Astra Pro, Better Font Awesome, BlogLentor, Editor clásico, Font Awesome viejo, HUSKY, Hustle, Kadence Security, Real3D Flipbook, reCAPTCHA Woo, ShopEngine, Smash Balloon, UAE, WhatsApp Chat NinjaTeam, WPForms Lite, Royal Elementor Addons + Pro, y especialmente WooCommerce Legacy REST API (riesgo de seguridad activo).

Instalar/dejar activos

PluginPara qué
Elementor + ProBuilder y Theme Builder (Header/Footer/Single)
WooCommerceCatálogo, fichas, carrito
JetEngineCustom fields para fichas técnicas (voltaje, RIN, etc.)
JetWooBuilderPlantillas Single Product + Archive
JetSmartFiltersFiltros AJAX en archive
Rank Math SEOSchema, sitemap, meta tags
TemaActivar Hello Elementor y desactivar su Header, Footer y Page Title nativos en Hello Elementor → Settings. Si tenían un tema previo (Astra, Royal), cambiarlo antes de cualquier edición o se rompen los estilos.

  font-size: 12px !important;

/* Page-wide */

  background: var(--surface-base);

}

/* Reduced motion */
@media (prefers-reduced-motion: reduce) {

}
Por qué !important en los botonesElementor inyecta inline styles desde el panel de cada widget. Sin !important tus clases globales se las come.

04Header sticky + glass on scroll

La parte donde Elementor más pelea. Aquí está la receta que funciona.

Estructura

Editar template Header — Global. Crear 1 Container flex horizontal con:

  • Container interno izquierda: widget Site Logo + Heading "Dimapar" (Outfit 800) + caption eyebrow.
  • Container interno centro: widget Nav Menu (Elementor Pro) apuntando al menú "Principal Nuevo" (crearlo antes en Apariencia → Menús).
  • Container interno derecha: Icon Button buscar + Icon Button carrito + widget Button WhatsApp con CSS Class btn-whatsapp.

Sticky en Elementor

Selecciona el container raíz del header → pestaña Advanced → Motion Effects:

  • Sticky: Top
  • Sticky On: Desktop + Tablet + Mobile
  • Effects Offset: 40 (cuándo se aplica la clase elementor-sticky--effects)
  • Z-Index: 50

Glass on scroll

Ahora la magia. En Advanced → CSS del mismo container, pega:

selector {
  background: transparent;
  transition: background .25s ease, backdrop-filter .25s ease;
}
selector.elementor-sticky--effects {
  background: rgba(10,14,26,0.70) !important;
  backdrop-filter: blur(20px) saturate(180%);
  -webkit-backdrop-filter: blur(20px) saturate(180%);
  border-bottom: 1px solid rgba(255,255,255,0.10);
}
selector es palabra reservadaEn el bloque CSS de Elementor, selector apunta automáticamente al widget/container actual. No lo cambies por una clase manual.

Mega menú "Productos"

Usar widget Nav Menu (Pro) con la opción nativa Menu Item → CSS Classes → mega-menu. Cada categoría como item con icono SVG inline + descripción + count dinámico (via JetEngine dynamic field si está configurado, o estático para MVP).

Alternativa más flexible: instalar JetMenu (Crocoblock plugin separado, verificar licencia) — soporta containers Elementor dentro del dropdown.

06Brand bar marquee

Esta es la sección más sencilla.

  • Section con background --surface-elevated, padding vertical 48px.
  • Heading H2 "Marcas oficiales que distribuimos" centrado, clase eyebrow.
  • Widget Image Carousel (Elementor Pro):
    • Slides per view: 5 desktop / 3 tablet / 2 mobile
    • Autoplay ON, speed 3000ms, infinite loop
    • Pause on hover ON
    • Navigation: ninguna (dots y arrows off — es decorativo)

Subir logos como PNG transparente blanco o SVG. CSS Custom para grayscale + hover color:

selector .swiper-slide img {
  filter: grayscale(1) opacity(0.7);
  transition: filter .25s;
}
selector .swiper-slide img:hover { filter: none; }

07Categorías destacadas (JetEngine Listing)

8 cards dinámicas. Una vez configurado no tocas nada — el día que agregues una categoría aparece sola.

Crear el Listing Item

  1. JetEngine → Listings → Add New
  2. Source: Terms, Tax: product_cat
  3. Diseñar el item (es como una mini-página Elementor):
    • Container con CSS Class glass-card, padding 24px, min-height 220px, link dinámico a "Term Page URL"
    • Icon (subir SVG por categoría como term meta o usar Icon widget con iconos fijos por slug — JetEngine permite condicionales)
    • Dynamic Field → name → HTML tag h3
    • Dynamic Field → description (term meta)
    • Dynamic Field → posts count, format {value} productos, con clase eyebrow

En la página Home insertar el grid

  • Widget Listing Grid (JetEngine) → seleccionar el listing creado
  • Columns: 4 desktop / 3 tablet / 2 mobile
  • Posts per page: 8
  • Order: by menu_order (controlas el orden manualmente desde la edición de cada categoría)
  • Equal columns height: ON

08Productos destacados

Mismo patrón que categorías pero apuntando a productos Woo con tag destacado-home.

  1. En Woo: crear tag destacado-home. Etiquetar 4–6 productos.
  2. Crear Listing Item JetEngine, source: WooCommerce Products. Diseñarlo replicando la card del mockup:
    • Container, background --surface-elevated (sólido, NO glass), border-radius 16px, overflow hidden
    • Container interno media: aspect-ratio 1:1, background #F8FAFC claro, image dinámica del producto
    • Badge "Destacado" condicional (mostrar si product_tag = destacado-hero)
    • Brand: dynamic field del atributo "Marca" con clase eyebrow
    • Nombre H3 Outfit 600
    • Specs: 3 chips con dynamic fields de atributos (Voltaje, RIN, etc.)
    • Footer con precio (font-family JetBrains Mono, tabular-nums) + 2 botones icon: ver ficha y WhatsApp
  3. Insertar Listing Grid en home, mode: Slider, 4 cards visible desktop / 1.2 mobile.

Botón WhatsApp con mensaje dinámico

El link de cada card debe incluir el nombre del producto. En el Button dinámico del listing:

URL: https://wa.me/593968663866?text=Consulta%20sobre%20{{product_name}}

Usar el módulo de dynamic tags de Elementor + el formato URL-encoded de JetEngine.

09Servicio técnico · Pilares · Catálogos

Estas tres son estáticas, ningún ítem se repite. Construir directo en la página, sin JetEngine.

Servicio técnico

  • Section, 2 columnas 50/50
  • Col izq: Container con clase glass-card glass-card--cyan, contenido en orden: eyebrow → Heading H2 → Icon List (5 items con iconos Lucide y descripción de 2 líneas) → 2 botones (cotización + WhatsApp)
  • Col der: Image widget con la foto del técnico que envíe el cliente. Border-radius 20px, aspect-ratio 1/1.

Pilares (Por qué Dimapar)

Section con background --surface-deep. Inner section con 4 columnas (2x2 en mobile). Cada columna es un widget Icon Box:

  • Icon position: top
  • Icon size: 40px, color cyan, fondo glass-cyan rectángulo (vía Advanced CSS)
  • Title: Outfit 600 18px
  • Description: Inter 400 14px color muted
  • Card background: --surface-elevated, border 1px --glass-border, padding 32px, radius 16px

El primer pilar usa el número "20+" en lugar de icono → reemplazar el Icon Box por un widget Heading dentro de un container con misma estética.

Catálogos PDF

  1. Opción dinámica: crear CPT "Catálogos" en JetEngine con campos: título, thumbnail, archivo PDF, peso, categoría. Mostrar con Listing Grid.
  2. Opción estática (MVP): Inner Section 3 columnas. Cada una: Image (cover PDF) + Heading + Text + Button "Descargar" linkeando al PDF subido a Media Library.
Nombre de archivo SEOSubir como dimapar-catalogo-alineadoras-hofmann-2026.pdf, no cat-1.pdf. Y agregar alt text al thumbnail.

10CTA asesor · Footer · WhatsApp flotante

CTA asesor

  • Section con ID cotizar (para que los botones href="#cotizar" hagan smooth scroll)
  • Container con clase glass-card glass-card--cyan, padding 64px
  • 2 columnas 60/40:
    • Izq: widget Form (Elementor Pro Forms). Campos: Nombre (required), WhatsApp tipo tel (required), Mensaje textarea. Acciones post-submit: Email a ventas@ + Redirect a página de gracias. Activar reCAPTCHA v3 en Site Settings.
    • Der: heading eyebrow + Heading número WhatsApp en mono 24px + Text "Respuesta en < 1h hábil" + Button verde grande clase btn-whatsapp + Icon List con teléfono, email, dirección.

Footer

  • Editar template Footer — Global.
  • Background --surface-deep, padding-top 80px, padding-bottom 32px.
  • Línea decorativa superior cyan: 1px height, gradient transparent → cyan-400 → transparent (widget Divider con custom CSS o un container vacío con background-image gradient).
  • 4 columnas (en mobile 1 col):
    1. Brand: Logo + tagline + descripción + 3 íconos sociales
    2. Productos: widget Nav Menu con menú "Footer Productos"
    3. Empresa: widget Nav Menu con menú "Footer Empresa"
    4. Contacto: Icon List con direccción, WhatsApp, email, horario
  • Footer bottom: Inner Section flex con copyright a la izq y links legales a la der.

WhatsApp flotante

No usar el plugin "WhatsApp Chat" que tenías — pesa demasiado. Hacerlo nativo:

  1. Editar el template Footer — Global.
  2. Agregar al final un Container con: Position fixed, Bottom 24px, Right 24px, Z-index 40.
  3. Adentro: widget Icon Box → icono WhatsApp 28px, fondo verde #25D366, círculo 60x60, sin texto.
  4. Link al https://wa.me/593968663866 con target _blank.
  5. CSS Custom para el pulse animado:
selector::before {
  content: "";
  position: absolute; inset: -6px;
  border-radius: 50%;
  border: 2px solid rgba(37,211,102,0.40);
  animation: dpulse 2.2s ease-out infinite;
}
@keyframes dpulse {
  0%   { transform: scale(0.9); opacity: 1; }
  100% { transform: scale(1.3); opacity: 0; }
}

11Performance + SEO

Imágenes

  • Instalar el plugin ShortPixel o Imagify. Configurar conversión automática a WebP + lazy load nativo (no JS).
  • Hero image: marcar como Loading: Eager en el widget Image. Resto en Lazy.
  • Tamaños WP: editar Settings → Media. Generar thumbnails 800w / 1600w para el hero.

Fuentes

  • Site Settings → Performance: activar Load fonts locally. Bajan los 3 Google Fonts al servidor y eliminan el roundtrip a fonts.googleapis.
  • Display: swap (Elementor lo aplica por default).

JS

  • Site Settings → Features: activar Improved Asset Loading y Optimized DOM Output.
  • Plugin extra recomendado: FlyingPress o Perfmatters para defer + remove unused CSS por página.

Schema.org

Rank Math → Schema → configurar:

  • Organization a nivel sitio (Rank Math lo hace al setup wizard).
  • WebSite con SearchAction (Rank Math automático si tienes buscador).
  • BreadcrumbList activar en Rank Math → General → Breadcrumbs.
  • Product en cada ficha: viene de WooCommerce, validar en Rich Results Test.

Sitemap + robots

  • Rank Math → Sitemap Settings → activar y forzar incluir productos + categorías.
  • Submit en Google Search Console.
  • robots.txt en raíz: NO bloquear /wp-content/uploads/.

12QA final · checklist de cierre

Antes de mostrar al cliente

  • Lighthouse en mobile + desktop. Objetivo: Performance ≥ 75 / A11y ≥ 95 / Best Practices ≥ 90 / SEO ≥ 95.
  • Probar el sticky header en Safari iOS — es donde más rompe.
  • Probar mega-menu con teclado (Tab + Enter + Esc).
  • Validar contraste con axe DevTools en Chrome.
  • Capturas a 375px, 1024px, 1440px del Home + Single Product + Archive.
  • Probar el formulario: que llegue el email a ventas@.
  • Probar WhatsApp en mobile real (no emulador) — el deep link wa.me a veces falla en webview.
  • Revisar que no aparezca el banner "powered by Elementor" en el footer.

Antes de poner en producción

  • Backup completo con UpdraftPlus (DB + archivos).
  • Si hay tráfico activo: hacer los cambios en un staging primero. Si no hay staging (es el caso de Dimapar): modo mantenimiento con Rank Math 503 page mientras editas.
  • Limpiar caché de WP, Cloudflare si aplica, y CDN.
  • Submit del sitemap actualizado en GSC y avisar a Google del cambio mayor.
